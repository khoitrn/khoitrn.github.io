My Learning

I’ve loved learning about IAM, cloud service models, and the AWS Shared Responsibility Model. IAM provides fine-grained control over who can access resources and what actions they can take. I practiced the differences between users, groups, and roles, and how JSON IAM policies implement least privilege.

We compared cloud service models: with IaaS, customers manage more (OS, runtime, apps) in exchange for maximum control. I also explored serverless (AWS Lambda)—often grouped with PaaS/FaaS—which reduces operational burden so I can focus on code, data, and permissions.

The Shared Responsibility Model clarified “security of the cloud” (AWS secures data centers, networking, hardware) versus “security in the cloud” (customers configure and secure what they run). I practiced encryption at rest/in transit, reviewed safe S3 access configurations, and saw how AWS Artifact and AWS Config support compliance and governance.
