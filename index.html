Your Learning

So far I loved learned about IAM, cloud service models, and the AWS Shared Responsibility Model. IAM provides fine-grained control over who can access resources and what actions they can take. I learned the differences between users, groups, and roles, and how JSON IAM policies implement the principle of least privilege.

We compared cloud service models: with IaaS, customers manage more (OS, runtime, apps) in exchange for maximum control. I also looked at serverless (AWS Lambda)—often grouped with PaaS/FaaS—which reduces operational burden by managing the underlying infrastructure so I can focus on code, data, and permissions.

The AWS Shared Responsibility Model clarified “security of the cloud” (AWS secures data centers, networking, hardware) versus “security in the cloud” (customers configure and secure what they run). I practiced securing data with encryption at rest and in transit, reviewed correct S3 bucket access configurations, and saw how services like AWS Artifact and AWS Config support compliance and governance.
