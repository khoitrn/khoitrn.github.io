My Learning

In this course, I gained practical experience with Identity and Access Management (IAM), learning how users, groups, and roles work together to manage permissions securely. I wrote and tested JSON policies that enforce the principle of least privilege to limit access based on roles.

Comparing cloud service models helped me understand the trade-offs between control and convenience. With IaaS, customers manage more components like the OS and runtime for greater flexibility, while serverless (AWS Lambda) reduces operational work, letting developers focus on code and data.

The AWS Shared Responsibility Model clarified what AWS secures (data centers, hardware, networking) versus what users secure (configuration, access, and data). I practiced encrypting data in transit and at rest, reviewed safe S3 access policies, and used AWS Config to monitor compliance.
